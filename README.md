# О чём репозиторий

Это тестовое cli приложение для обработки csv файлов с метриками YouTube видео и формированием отчётов по заданным параметрам.

# Что используется

Из сторонних библиотек в проекте используется pytest, tabulate, ruff, *setuptools. 

> setuptools устанавливается для сборки с pyproject.toml. Изначально я подумал отказаться от установки проекта как пакета из-за требования использовать только стандартные библиотеки, однако в FAQ указано, что можно *использовать что-то кроме pip для управления зависимостями*. Считаю, что данный кейс подходит, и проект так выглядит приятнее.

# Общая логика

`cli.py` - точка входа. В нём вызывается `app.run()` - оркестрация функционала.
`VideoMetric` - класс для строк из CSV файла. 
Через `get_report(report_name)` передаётся название отчёта, происходит поиск в папке reports класса отчёта с именем `report_name`. Если успешно находит, возвращает объект этого класса в `app.run()`. Дальше идёт чтение CSV, конвертация в список `VideoMetric` и составление отчёта по условиям, которые описаны в классе отчёта.


## Добавление новых отчётов

Для добавления новых отчётов требуется создать новый файл в папке reports и описать в нём класс отчёта. Менять какие-либо файлы не требуется.

> В не функциональных требованиях ТЗ указано, что возможность добавления отчётов должна быть заложена в `--report`. Однако, очевидно, что для добавления новых отчётов исключительно через параметр названия отчёта недостаточно. Требуется дополнительные параметры для описания условий этого отчёта, а это, насколько я понимаю, запрещено ТЗ. Была идея либо полностью отказаться от кода отчётов ( в командной строке описывать условия отчёта), либо использовать дополнительный флаг `--add`.

# Установка

C тестами и ruff.
```
pip install -e .[dev]
```

# Примеры запуска

```

```
